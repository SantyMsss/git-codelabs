¿Qué es Git y GitHub?

Git es un sistema de control de versiones distribuido que te permite rastrear cambios en tus archivos a lo largo del tiempo. Es especialmente útil para desarrolladores que trabajan en equipo.

Git vs GitHub

  • Git: Es la herramienta de control de versiones que se ejecuta localmente en tu computadora.
  • GitHub: Es una plataforma en la nube que aloja repositorios Git y facilita la colaboración.

Conceptos básicos

  • Repositorio: Un proyecto de Git que contiene todos los archivos y su historial.
  • Commit: Un conjunto de cambios guardados en el historial del repositorio.
  • Rama (Branch): Una línea de desarrollo independiente dentro del repositorio.
  • Merge: Combinar cambios de diferentes ramas.

Crear una cuenta en GitHub

GitHub es la plataforma más popular para alojar proyectos con Git. Sigue estos pasos para crear tu cuenta:

  1. Ve a github.com
  2. Haz clic en "Sign up"
  3. Ingresa tu correo electrónico
  4. Crea una contraseña segura
  5. Elige un nombre de usuario
  6. Verifica tu cuenta a través del correo electrónico

Simulación de registro

¡Cuenta creada exitosamente! Ahora puedes continuar con el siguiente paso.

Configurar Git en tu equipo

Antes de usar Git, necesitas configurar tu identidad. Esto es importante porque cada commit que hagas estará asociado a esta información.

Instalación de Git

Si no tienes Git instalado, descárgalo desde git-scm.com y sigue las instrucciones de instalación para tu sistema operativo.

Configuración inicial

Abre tu terminal o línea de comandos y ejecuta los siguientes comandos (reemplaza con tu información):

$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tu@email.com"

Configura tu identidad

¡Git configurado correctamente! Ahora puedes continuar.

Crear tu primer repositorio

Un repositorio es donde Git almacena todos los archivos y el historial de cambios de tu proyecto.

Crear un repositorio local

Para crear un nuevo repositorio, abre tu terminal y navega a la carpeta donde quieres almacenar tu proyecto, luego ejecuta:

$ mkdir mi-primer-proyecto
$ cd mi-primer-proyecto
$ git init
Initialized empty Git repository in /ruta/a/mi-primer-proyecto/.git/

Crear un repositorio en GitHub

También puedes crear un repositorio en GitHub y luego clonarlo localmente:

  1. Inicia sesión en GitHub
  2. Haz clic en el icono "+" en la esquina superior derecha
  3. Selecciona "New repository"
  4. Asigna un nombre a tu repositorio
  5. Elige si será público o privado
  6. Haz clic en "Create repository"

Simulación de creación de repositorio

¡Repositorio creado exitosamente! Ahora puedes continuar.

Hacer tu primer commit

Un commit es como una "fotografía" de los cambios en tu proyecto en un momento específico. Para hacer un commit, sigue estos pasos:

  1. Crea o modifica archivos en tu proyecto
  2. Añade los archivos al área de preparación (staging)
  3. Realiza el commit con un mensaje descriptivo
$ echo "# Mi Primer Proyecto" >> README.md
$ git add README.md
$ git commit -m "Agregar archivo README"
[main (root-commit) abc1234] Agregar archivo README
1 file changed, 1 insertion(+)
create mode 100644 README.md

Flujo de trabajo básico

  • Working Directory: Donde realizas cambios en tus archivos
  • Staging Area: Donde preparas los archivos para el commit
  • Repository: Donde se almacenan los commits de forma permanente

Simulación de commit

¡Commit realizado exitosamente! Los cambios han sido guardados en el historial.

Subir cambios a GitHub

Para compartir tu trabajo con otros o tener una copia de seguridad en la nube, puedes subir tu repositorio local a GitHub.

Conectar repositorio local con GitHub

Si creaste tu repositorio localmente, necesitas conectarlo con un repositorio remoto en GitHub:

$ git remote add origin https://github.com/tu-usuario/mi-primer-proyecto.git
$ git branch -M main
$ git push -u origin main
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Writing objects: 100% (3/3), 245 bytes | 245.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0)
To https://github.com/tu-usuario/mi-primer-proyecto.git
* [new branch] main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.

Subir cambios futuros

Una vez configurado, para subir cambios futuros solo necesitas:

git add .
git commit -m "Descripción de los cambios"
git push

¡Felicidades!

Has completado el tutorial de introducción a Git y GitHub. Ahora tienes los conocimientos básicos para:

  • Crear y gestionar repositorios
  • Realizar commits de tus cambios
  • Colaborar con otros mediante GitHub

Continúa practicando y explorando más funcionalidades de Git como las ramas (branches), fusiones (merges) y solicitudes de extracción (pull requests).
